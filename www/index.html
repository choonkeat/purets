What is it?

.pure.ts files are valid TypeScript with one constraint: everything must be pure. No side effects, no IO, no mutation — just types, data, and functions.

Think of it as "use strict" for purity. Your editor already understands it. tsc already type-checks it. purets just enforces the rules.

// models.pure.ts

type User = {
  name: string
  age: number
}

type Status = "active" | "inactive"

function greet(user: User): string {
  return "Hello " + user.name
}

const adults = (users: User[]): User[] =>
  users.filter(u => u.age >= 18)

const alice: User = { name: "Alice", age: 30 }

The Rules

Allowed

  • type declarations
  • const values
  • function declarations
  • Arrow functions
  • Named import from .pure.ts
  • Named export
  • Generics, unions, tuples

Banned

  • class, interface, enum
  • let, var
  • this
  • async / await
  • console, fetch, process
  • export default
  • Import from non-.pure.ts
  • Return void, any, Promise

Quick Start

npm install purets

Check a file

purets check data.pure.ts

Pass extra tsc flags

purets check data.pure.ts -- --noUnusedLocals

Launch the web editor

purets edit
purets edit ./data --port 8080

Pure Islands

You don't have to convert your whole project. Use .pure.ts for the parts that benefit from purity guarantees:

src/
  api/
    routes.ts — Express handlers, IO
    middleware.ts — side effects
  domain/
    models.pure.ts — types + constructors
    validators.pure.ts — pure validation
    transforms.pure.ts — pure transforms
  utils/
    math.pure.ts — pure utilities
    format.pure.ts — pure formatting
  index.ts — wires everything together

Regular .ts files can import from .pure.ts freely. The purity guarantee flows one way: pure code can't depend on impure code, but impure code can use pure modules.

How It Works

purets uses the TypeScript compiler API directly. No custom parser, no transpilation, no runtime.

1. AST Validation

Walks the syntax tree to enforce subset rules — detects banned constructs like class, this, console.

2. Type Checking

Runs tsc --strict via the compiler API for full type safety.

3. Return Type Analysis

Checks inferred return types — catches void and any even without explicit annotations.

Web Editor

purets edit launches a browser-based editor powered by Monaco (same editor as VS Code). Live type checking, collapsible file tree, mobile responsive.
